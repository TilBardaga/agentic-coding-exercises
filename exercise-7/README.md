# Exercise 7: Review commands (code and system)

Exercise 7 combines **technical code review** and **meta-level system review** in two steps, numbered **7.1** and **7.2**. In the suggested workshop order, do **[exercise 6](../exercise-6/README.md)** ( **`/execute`** ) after planning so you have real changes to review; exercise 7 then tightens quality and process.

## Overview

1. **Exercise 7.1 — Code review**  
   Build a command that reviews **changed code** for bugs, security, and quality—aligned with `AGENTS.md` and project patterns.

2. **Exercise 7.2 — System review**  
   After you capture an **execution report** (what actually happened vs the plan), build a command that reviews **process** and suggests updates to Layer 1 assets and templates.

Do **7.1** before **7.2** if you want the narrative “technical review → then meta review,” but they can be done in either order.
